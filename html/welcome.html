Vokabeltrainer s74053

Dieser Vokabeltrainer ist die Belegaufgabe des Moduls „Internet-Technologien 1“ vom Sommersemester 2017.

Es wurden folgende Anforderungen aus der Aufgabenstellung erarbeitet, die aus den gekennzeichneten Quellen abgeleitet wurden:

  • Anforderung … Anforderung ergibt sich aus Aufgabentext
  • Anforderung … Anforderung ergibt sich aus dem Aufgabentext angehängten Bildern
  • Anforderung … Anforderung ergibt sich implizit / wurde eigenverantwortlich festgelegt

Allgemeine Funktionalität:

  • Vokabellisten liegen in einer Textdatei im UTF-8 Format vor. Jede Zeile enthält per Tab getrennt „Sprache 1“, „Sprache 2“ und ggf. „Aussprache“. In dieser Implementation hat die Textdatei demnach folgende Struktur:
    Wort in Sprache 1 ↦ [Aussprache Wort in Sprache 1] ↦ Übersetzung in Sprache 2
  • Vokabellisten können per Fileupload über den Bereich Setup hinzugefügt werden. Es dürfen nur .txt-Dateien hoch geladen werden, die keine HTML-Tags haben. Es dürfen nur Dateien hochgeladen werden, die dem Schema entsprechen (Prüfkriterium: 2 Tabs / 3 Einträge pro Zeile)
    Optional: Bearbeiten der (hochgeladenen) Vokabellisten.
  • Übergabe von Werten erfolgt über POST (bei Ajax wird GET verwendet).
  • Für die Navigation (Lektion, Statistik, …) wird GET verwendet. Die Navigationsleiste befindet sich am unteren Rand. Aktive Elemente werden ausgeblendet.
  • Die aktive Lektion/Seite wird in einer Titelleiste und als HTML-Meta-Information gesetzt.
  • Es wird jQuery Mobile verwendet.
  • Es wird auf ein responsive Design geachtet. Die Felder für die Vokabelabfrage (Gefragte Vokabel, Auswählfeld) werden je nach Breite des Browser-Fensters untereinander oder nebeneinander auf der vollen Breite dargestellt.
  • Es wird HTML, PHP5, Javascript, CSS und Ajax verwendet.
  • Das Ergebnis muss unter Mozilla Firefox lauffähig sein. Es wird nur auf Desktop-Browsern getestet. Internetverbindung muss vorhanden sein.
  • Die Seite ist auf eine Bedienung durch einen einzelnen Nutzer ausgelegt. Es existiert keine Nutzereverwaltung. Es wird davon ausgegangen, dass nur ein Nutzer gleichzeitig auf die Dateien (Vokabellisten/Statistik) zugreift. Fehler durch mehrfachen (Schreib-)Zugriff werden nicht behandelt.
  • Leere Eingaben und mögliche Fehlerzustände werden abgefangen.
  • Der Code wird verständlich und sinnvoll kommentiert. Da der CSS Code teilweise aus einer eigens erstellten Vorlage besteht, sind nur die Elemente, die explizit für diesen Beleg erstellt wurden besonders kommentiert.

Seitenspezifische Anforderungen:

Vokabelabfrage:

  • Bei dem Vokabeltest werden 5 unterschiedliche Übersetzungen angezeigt, die Auswahl erfolgt per Radio-Button (Single Choice). Keine Antwort gilt als falsche Antwort. Mit einem Button „senden“ wird die Auswahl bestätigt.
  • Die Abfragerichtung soll während des Tests geändert werden. Dies soll ohne neu laden geschehen (⇒ Ajax). Die aktive Abfragerichtung soll durch die Elemente für die Änderung der Abfragerichtung visualisiert sein.
  • Beim Absenden der Antwort erscheint die Ergebnisseite, wo richtige/falsche Antworten grün/rot hinterlegt werden. Es wird die Abgefragte Vokabel, ggf. die entsprechende Lautschrift und die richtige Antwort angezeigt (bei falscher Antwort, wird die falsche Antwort durchgestrichen vor der richtigen dargestellt). Mit der Schaltfläche „weiter“ geht es zur nächsten Abfrage.
  • Es ist nicht gefordert, dass eine „Session“ existieren soll, in der gespeichert werden, welche Vokabeln in dieser bereits abgefragt wurden. Daher wird die nächste Vokabel nach dem Beantworten zufällig ausgewählt (und kann sich im Ernstfall wiederholen).

Stastik:

  • Für jede Vokabelliste (Datei) wird eine Statistik in einer dazugehörigen Datei geführt. Existiert die Datei nicht (mehr), wird auch keine Statistik (mehr) für sie angezeigt.
  • Die Statistik besteht aus der Anzahl der richtigen/falschen Antworten.
  • Der Bereich Statistik listet die Statistiken für alle (aktive) Vokabelliste auf.

Lektionsauswahl

  • Alle vorhandenen Lektionen/Dateien werden aufgelistet.
  • Die aufgelisteten Elemente sind nach den Dateien benannt.

Setup-Seite

  • Die Setup-Seite besteht aus einem „Datei hochladen“ Feld (im einfachsten Fall).
